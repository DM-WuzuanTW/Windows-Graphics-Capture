## 專案進度總覽

### ✅  已完成的工作

1. **核心功能修復與優化**
   - ✅ **修復 GPU 崩潰**: 在 `main.js` 中禁用硬體加速，解決了 Electron 的 GPU 進程崩潰問題。
   - ✅ **修復 Windows API 整合**: 
     - 修正 `EnumWindows` 回調函數的返回值類型（從 1 改為 true）。
     - 加入 `GetLastError` 以獲取詳細錯誤代碼。
   - ✅ **增強錯誤處理**: 
     - `ExclusionManager` 現在能識別權限不足 (Error 5) 和無效視窗 (Error 1400)。
     - 提供更友好的錯誤訊息給前端。
   - ✅ **狀態管理優化**: 防止對已排除的視窗重複操作，並正確處理視窗關閉的情況。

2. **UI 優化**
   - ✅ **性能優化**: 在 `renderer.js` 中加入 hash 檢查，避免視窗列表無變化時的 DOM 重繪。
   - ✅ **錯誤反饋**: 前端現在能顯示具體的錯誤原因（如「權限不足」）。

### 📋 當前狀態

- **應用程式穩定性**: 高。已解決主要的崩潰和錯誤。
- **功能完整性**: 
  - 視窗列舉：正常
  - 視窗排除：正常（受限於權限）
  - 設定保存：正常
- **已知限制**: 
  - 無法排除以管理員身分運行的程式視窗（如 Task Manager, Admin CMD），這是 Windows 安全機制 (UIPI) 的限制。應用程式會提示權限不足。

### 🚀 後續建議

1. **權限提升提示**: 考慮添加一個功能，讓使用者能以管理員身分重啟應用程式，以獲得更高的權限。
2. **更多過濾選項**: 允許使用者過濾掉系統進程或無標題視窗。
3. **托盤圖示**: 實作系統托盤圖示，讓應用程式可以在後台運行。
